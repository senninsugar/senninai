## 1. トークナイザー (Tokenizer)

### 目的
テキストを機械学習で処理できる数値表現に変換

### 処理フロー

```
入力テキスト
    │
    ▼ 1. 前処理
    └─ 小文字に統一
    └─ 特殊文字を削除
    └─ スペースで分割
    
    ▼ 2. トークン化
    "こんにちは今日は天気がいいですね"
        ↓
    ["こんにちは", "今日", "は", "天気", "が", "いいです", "ね"]
    
    ▼ 3. ID変換
    単語辞書を参照して、各単語をIDに変換
    
    "こんにちは" → 142
    "今日"      → 89
    "は"        → 234
    ...
    
    ▼ 4. 出力
    Token IDs: [142, 89, 234, ...]
```

### 語彙の構築

```javascript
// 学習時
const texts = [
    "こんにちは",
    "今日はいい天気",
    "プログラミングは楽しい"
];

// 単語の出現頻度をカウント
Word Freq.
"は"      10
"いい"     8
"今日"     5
"天気"     3
"こんにちは" 2
...

// 頻度順にソートして、ID を割り当て
0: <PAD>        (パディング用)
1: <UNK>        (未知語用)
2: <START>      (開始記号)
3: <END>        (終了記号)
4: "は"         (最頻出)
5: "いい"
6: "今日"
...
```

## 2. エンベディング層 (Embedding Layer)

### 目的
離散的なToken IDを、連続的な密ベクトル（埋め込み）に変換

### 変換プロセス

```
Token ID    Embedding Vector (128次元)
    │
    0 (PAD)   → [0.0,   0.1,  -0.2,  0.3,  ...]
    142       → [0.5,  -0.1,   0.8, -0.4,  ...]  ← "こんにちは"の意味表現
    89        → [-0.2,  0.6,  -0.1,  0.2,  ...]  ← "今日"の意味表現
    234       → [0.1,   0.3,   0.4, -0.5,  ...]  ← "は"の意味表現
```

### ベクトルの解釈

各次元は単語の特徴を表す（学習中に自動決定）

```
例: "今日"の埋め込み
[ 0.8, -0.2,  0.5, 0.1, -0.3, ...]
  ▲     ▲      ▲
  │     │      └─ 「天気に関連」の度合い
  │     └────── 「過去/現在/未来」の度合い
  └─────────── 「時間概念」の度合い
```

### 実装

```javascript
class EmbeddingLayer {
  constructor(vocabSize = 5000, embedDim = 128) {
    // (5000 × 128) の重み行列を初期化
    // 各行が1つの単語の埋め込みベクトル
    this.embeddings = [
      [0.5, -0.1, 0.8, ...],  // Token 0
      [0.2,  0.3, -0.1, ...], // Token 1
      ...
    ]
  }
  
  embed(tokenId) {
    // embeddings[tokenId] を返す
    return this.embeddings[tokenId];
  }
  
  embedSequence(tokenIds) {
    // 複数のトークンを一度に埋め込む
    return tokenIds.map(id => this.embed(id));
  }
}
```

## 3. 位置エンコーディング (Positional Encoding)

### 目的
トークンの順序情報を埋め込みに追加

**なぜ必要か:**
Transformerは各トークンを平行処理するため、位置情報が失われる
→ 位置情報を明示的に追加する必要がある

### サイン・コサイン位置エンコーディング

```
PE(pos, 2i)     = sin(pos / 10000^(2i/d_model))
PE(pos, 2i+1)   = cos(pos / 10000^(2i/d_model))

例: pos=0, d_model=128
PE_0 = [sin(0/10000^0),   cos(0/10000^0),
        sin(0/10000^1),   cos(0/10000^1),
        sin(0/10000^2),   cos(0/10000^2),
        ...
       ]
    = [0.0, 1.0, 0.0, 1.0, 0.0, 1.0, ...]

例: pos=1, d_model=128
PE_1 = [sin(1/10000^0),   cos(1/10000^0),
        sin(1/10000^1),   cos(1/10000^1),
        ...
       ]
    = [0.841, 0.540, 0.0999, 0.995, ...]
```

### 利点

- **ユニークな周期性**: 各位置が異なるベクトルを持つ
- **相対位置を保持**: 位置の差は学習で理解可能
- **長シーケンスに対応**: 理論上無限長に対応可能

### 実装

```javascript
class PositionalEncoding {
  generatePositionalEncoding(maxLen, dim) {
    const pe = [];
    for (let pos = 0; pos < maxLen; pos++) {
      const row = [];
      for (let i = 0; i < dim; i++) {
        if (i % 2 === 0) {
          row.push(Math.sin(pos / Math.pow(10000, i / dim)));
        } else {
          row.push(Math.cos(pos / Math.pow(10000, (i - 1) / dim)));
        }
      }
      pe.push(row);
    }
    return pe;
  }
  
  addPositionalEncoding(embeddings) {
    return embeddings.map((emb, pos) => 
      emb.map((val, i) => val + this.pe[pos][i])
    );
  }
}
```

## 4. マルチヘッド注意 (Multi-Head Attention)

### 目的
トークン間の関係性を学習

**類似概念:**
- 人間が文を読む時、複数の視点から理解する
- 例: "銀行" という単語は
  - 金融機関としての意味
  - 川岸としての意味
  を文脈で使い分ける

### 単一ヘッド注意の仕組み

```
入力シーケンス: "今日 は 天気 が いい"
              [x1, x2, x3, x4, x5]

ステップ1: Query, Key, Value に変換
┌─────────────────────────────────────┐
│ Query = X · W_q                     │
│ Key   = X · W_k                     │
│ Value = X · W_v                     │
│                                     │
│ (線形変換で異なる部分空間に投影)    │
└─────────────────────────────────────┘

ステップ2: 注意スコアを計算
         Query · Key^T
         
例: "天気" (x3) が他のトークンにどの程度注目するか？
         
"天気" の Query
    ↓
    ・ "今日" の Key → スコア = 0.2
    ・ "は" の Key   → スコア = 0.1
    ・ "天気" の Key → スコア = 0.9  ← 自分自身に高い注目
    ・ "が" の Key   → スコア = 0.3
    ・ "いい" の Key → スコア = 0.5

ステップ3: 注意重みを計算
softmax([0.2, 0.1, 0.9, 0.3, 0.5])
= [0.08, 0.07, 0.54, 0.13, 0.18]
     ▲    ▲    ▲    ▲    ▲
     └── "天気"は自分に54%、"いい"に18%...注目

ステップ4: Value に注意重みを適用
出力 = 0.08·Value1 + 0.07·Value2 + 0.54·Value3 + ...

→ "天気"の周辺文脈を統合した表現が得られる
```

### マルチヘッド注意

複数の注意メカニズムを同時に実行

```
入力 X (5 tokens × 128 dim)
    │
    ├─ Head 1: Query1, Key1, Value1 (4ヘッドで32次元ずつ)
    │          ↓ Attention ↓
    │          出力1 (5 tokens × 32 dim)
    │
    ├─ Head 2: Query2, Key2, Value2
    │          ↓ Attention ↓
    │          出力2 (5 tokens × 32 dim)
    │
    ├─ Head 3: Query3, Key3, Value3
    │          ↓ Attention ↓
    │          出力3 (5 tokens × 32 dim)
    │
    └─ Head 4: Query4, Key4, Value4
               ↓ Attention ↓
               出力4 (5 tokens × 32 dim)
    
    並列実行 → 異なる観点から同時に関係性を学習!

    ↓ 連結 (Concatenate)
    
    [出力1 | 出力2 | 出力3 | 出力4] (5 tokens × 128 dim)
    
    ↓ 線形変換
    
    最終出力 (5 tokens × 128 dim)
```

### 異なるヘッドが学習する例

```
Head 1: "天気" ← 形容詞 "いい" に注目
Head 2: "は"   ← 主語 "今日" と述語に注目  
Head 3: "今日" ← 時間関連情報に注目
Head 4: 全体   ← 文全体の構造に注目

→ 複雑な言語構造を多角的に理解!
```

  }
  
  forward(x) {

## 5. フィードフォワード層 (Feed-Forward Network)

### 目的
非線形変換で複雑な特徴を学習

### 構造

```
入力 (128-dim)
    ↓
  W_1 (128 × 512)  ← 拡大する
    ↓
 隠れ層 (512-dim)
    ↓
  ReLU 活性化関数 ← 非線形性を導入
    ↓
  W_2 (512 × 128)  ← 元に戻す
    ↓
出力 (128-dim)

公式: FFN(x) = ReLU(x·W_1 + b_1)·W_2 + b_2
```

### ReLU 活性化関数

```
ReLU(x) = max(0, x)

グラフ:
     │  /
     │ /
─────┼─────
    /│
   / │
  x=0で折れ曲がる
  
利点:
- 計算が高速
- グラディエント消失問題を緩和
- スパース性 (0を多く出力)
```

### 実装

```javascript
class FeedForward {
  constructor(embedDim, ffDim = 512) {
    this.w1 = (embedDim × ffDim);      // 拡大層
    this.w2 = (ffDim × embedDim);      // 縮小層
    this.b1 = new Array(ffDim).fill(0);
    this.b2 = new Array(embedDim).fill(0);
  }
  
  forward(x) {
    const hidden = x·w1 + b1;  // 線形変換
    const relu = hidden.map(v => Math.max(0, v));  // 活性化
    const output = relu·w2 + b2;  // 線形変換
    return output;
  }
}
```

## 6. Transformer ブロック

### 統合図

```
入力 X (5 tokens × 128 dim)
    │
    ▼
┌──────────────────────────┐
│ Multi-Head Attention     │
│ Att(X) = (5 tokens × 128)│
└────────────┬─────────────┘
             │
    ┌────────▼─────────┐
    │ Residual + Norm  │
    │ LN(X + Att(X))   │
    └────────┬─────────┘
             │
             ▼
    ┌──────────────────────────┐
    │ Feed-Forward Network     │
    │ FFN(X) = (5 tokens × 128)│
    └────────┬─────────────────┘
             │
    ┌────────▼─────────┐
    │ Residual + Norm  │
    │ LN(X + FFN(X))   │
    └────────┬─────────┘
             │
             ▼
       出力 (5 tokens × 128)
```

### 残差接続の重要性

```
入力 x について、f(x) を学習するのではなく、
残差 (x - f(x)) を学習!

利点:
1. グラディエント消失を防止
   → 深いネットワークでも学習可能
2. 短いパスで勾配が流れる
   → 安定した学習
3. スキップ接続で情報が失われない
   → より多くの情報を保持
```

### レイヤー正規化の役割

```
入力: [0.5, 2.3, -1.2, 0.1, ...]

正規化前:
 - 平均: 0.3
 - 分散: 1.2

正規化処理:
1. 平均を引く: [-0.2, 2.0, -1.5, -0.2, ...]
2. 標準偏差で割る: [-0.18, 1.82, -1.36, -0.18, ...]

正規化後:
 - 平均: 0
 - 分散: 1

効果:
- ニューラルネットの学習を安定化
- 内部共変量シフト (internal covariate shift) を削減
- より高い学習率を使用可能
```

## 7. 出力投影層と確率計算

### プロセス

```
最後の Transformer ブロックの出力
         │
         ▼
    シーケンスの最後のトークンのみを抽出
    (128-dim ベクトル)
         │
         ▼
    出力投影層 (128 × 5000)
         │
         ▼
    Logits (5000-dim)
         │
         ▼
    Softmax
         │
         ▼
    確率分布 (5000個の確率, 合計=1.0)
         │
         ▼
    サンプリング (温度パラメータ適用)
         │
         ▼
    次の単語 ID を選択
```

### Softmax 関数

```
softmax(x_i) = exp(x_i) / Σ exp(x_j)

例: 3つの単語の Logits = [2.0, 1.0, 0.1]

exp([2.0, 1.0, 0.1]) = [7.39, 2.72, 1.10]
合計 = 11.21

softmax = [7.39/11.21, 2.72/11.21, 1.10/11.21]
        = [0.66, 0.24, 0.10]

解釈:
- "天気"       → 66% の確率
- "くもり"     → 24% の確率  
- "雨"        → 10% の確率
```

### 温度スケーリング

```
Logits = [2.0, 1.0, 0.1]

温度 T = 0.5 (低温 = 決定的)
Logits/T = [4.0, 2.0, 0.2]
softmax → [0.95, 0.04, 0.01]  ← 最大値が支配的

温度 T = 1.0 (通常)
Logits/T = [2.0, 1.0, 0.1]
softmax → [0.66, 0.24, 0.10]

温度 T = 2.0 (高温 = ランダム)
Logits/T = [1.0, 0.5, 0.05]
softmax → [0.38, 0.33, 0.29]  ← より均等
```

## 計算量分析

### Time Complexity

```
Input: N tokens, d dimensions, h heads

Attention層:
- Query, Key, Value 計算: O(N·d²)
- 注意スコア計算: O(N²·d)
- Output投影: O(N·d²)
- 合計: O(N·d²) + O(N²·d)

FeedForward層:
- 第1層: O(N·d·d_ff)  where d_ff = 2048
- 第2層: O(N·d_ff·d)
- 合計: O(N·d·d_ff)

Transformer Block (1層):
O(N·d²) + O(N²·d) + O(N·d·d_ff)
≈ O(N²) (長いシーケンスでは支配的)

2層の場合:
O(2·N²) = O(N²)
```

### Space Complexity

```
重み行列:
- Attention: (d × d) × 4 = 4d² パラメータ
- FeedForward: (d × d_ff) + (d_ff × d) ≈ 2d·d_ff
- 合計: 4d² + 2d·d_ff

2層 × N語彙:
- Embedding: vocab × d = 5000 × 128 ≈ 640K
- Attention × 2: 4 × (128)² × 2 ≈ 131K
- FeedForward × 2: 2 × 128 × 512 × 2 ≈ 262K
- Output: d × vocab = 128 × 5000 ≈ 640K

合計: ≈ 1.6M パラメータ ≈ 6.4 MB (float32)
```

## 学習アルゴリズム

### 教師あり学習

```
訓練データ: [(x_1, y_1), (x_2, y_2), ...]

例:
入力:  "こんにちは 今日 は"
→ Token IDs: [142, 89, 234]

目標ラベル: 89 ("今日") の次は 234 ("は")

1. Forward Pass:
   - [142, 89, 234] をモデルに入力
   - 最後の位置での出力ベクトル (128-dim)
   - 5000次元の確率分布に変換
   - P("は") = 0.8

2. Loss計算 (Cross-Entropy):
   Loss = -log(P("は"))
        = -log(0.8)
        = 0.223

3. Backward Pass:
   - 勾配を逆伝播
   - 各パラメータを更新
   - W = W - α·∇Loss

4. 繰り返し:
   全訓練データでエポック分実行
```

### 最適化の課題と対策

```
課題 1: グラディエント消失
原因: 連鎖律により、層が深いと勾配が指数関数的に小さくなる
対策: 残差接続で短いパスを作成

課題 2: 内部共変量シフト
原因: 層ごとに入力分布が変わる
対策: レイヤー正規化で各層の入力を正規化

課題 3: 過学習
原因: モデルが訓練データに過度に適応
対策: ドロップアウト (未実装) や正則化
```

```

テキスト
  ↓

  ↓
Softmax (正規化)
  ↓
サンプリング
  ↓
次の単語を生成

このプロセスを繰り返して、長いテキストを生成!
```
